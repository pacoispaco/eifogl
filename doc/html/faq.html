EiffelOpenGL FAQ

1. What is EiffelOpenGL?

EiffelOpenGL is the collective name for EGL, EGLUT and EGLAUX.
  • EGL stands for Eiffel OpenGL and is an Eiffel encapsulation of OpenGL (ie. the GL and GLU libraries).
  • EGLUT stands for Eiffel GLUT and is an Eiffel encapsulation of GLUT (OpenGL Utility Toolkit library).
  • EGLAUX stands for Eiffel OpenGL AUX and is an Eiffel encapsulation of the OpenGL AUX (Auxillary) library.

2. What inplementations of OpenGL, GLUT and GLAUX are supported?

Currently only the Microsoft Win32 OpenGl implementation and the GLUT 3.6 (http://www.cs.utah.edu/~narobins/glut.html) implementation for Windows NT & 95. My goal is of course to have EGL run smoothly on all important OpenGL and GLUT implementations.
 

3. What do I need to use EiffelOpenGL?

To use EiffelOpenGL you need:
  • The ISE Eiffel compiler and IDE. EiffelOpenGL was developed with ISE 4.2f, but seems to work fine with ISE 4.5. At the moment EiffelOpenGL has some dependencies on WEL (ISE Eiffels Windows Eiffel Library). These will be removed in the near future. Since the semantics of the manifest string in the language part of the Eiffel external clause isn't covered by ETL or defined by NICE, it's interpretation depends upon what Eiffel compiler you are using.
  • To have installed the gl.h and glu.h include files and set your include path so that your C compiler can find them.
  • To have installed the opengl32.lib and glu32.lib import libraries, as well as the opengl32.dll and glu32.dll dynamic libraries.
The Microsoft Win32 OpenGL implementation is for instance shipped with Developer Studio 5.0 and 6.0. To use EGLUT you need, in addition to the above mentioned:
  • To have installed the glut.h include file and set your include path so that your C compiler can find it.
  • To have installed the glut32.lib import library as well as the glut32.dll dynamic library.
  • Compiled the eglut.lib (static) library.
To use EGLAUX you need, in addition to that needed for EiffelOpenGL:
  • To have installed the glaux.h include file and set your include path so that your C compiler can find it.
  • To have installed the glaux.lib library.

4. With which Eiffel compilers/IDEs and on which platforms does it work?

At the moment it has only been tested with ISE bench 4.2f, 4.3 and 4.5 on Windows 95/NT. I hope to be able to port it to SmallEiffel and X Window/Linux soon.
 

5. What is the current version of EiffelOpenGL?

0.1.0. Version numbering follows the format [Major Version].[Minor Version].[Patch Level].
 

6. How large is the EiffelOpenGL source code?

Currently it is comprised of 128 Eiffel classes, 2 C text files and 11 C header files.
 

8. Why should I use EiffelOpenGL?

Well, you can write your OpenGL/GLUT program in Eiffel! That means design-by-contract, carbage collection (if you want it turned on that is), multiple inheritance and readable syntax among other things. EiffelOpenGL attempts to use all the benefits of Eiffel's support for design-by-contract. This enables a lot of obscure or implicit conditions in the OpenGL and GLUT API can be implemented in formal contracts and class invariants and thus be made explicit.

11. What are the future development goals of EiffelOpenGL?

In no particular order:
  • Ensure platform portability. Primarily Win32 API and X Window. Maybe MacOS.
  • Ensure independence of WEL.
  • Continue the encapsulation work on OpenGL.
  • Continue the encapsulation work on GLUT.
  • Drop the encapsulation of GLAUX. GLAUX seems to be unsupported and is not being further developed.
  • Develop more comprehensive example programs. Evaluate performance of EiffelOpenGL.

12. Can I help with developing EiffelOpenGL?

Yes you certainly can. Either by just trying out the example programs and or using EiffelOpenGL and finding bugs and reporting them back to me (paco@enea.se) or by helping to continue writing the encapsulating code, or by testing and porting the code to other platforms and compilers. Or by sending me copious amounts of money or other useful things like a National Style N Reso-Phonic guitar. Just be sure to send me style guide formatted and compilable code additions or examples and readable and understandable suggestions for code changes.
